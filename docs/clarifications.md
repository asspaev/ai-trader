## 5. LLM и эмбеддинги

### 5.1. Какие модели через OpenRouter для агентов?

- a) **[рекомендую]** По умолчанию `anthropic/claude-haiku-4.5` для всех 3 агентов (PRICE/NEWS/TRADER) — быстро, дёшево, достаточно качественно. Модель настраивается отдельно на каждого агента через `.env` (`PRICE_AGENT_MODEL`, `NEWS_AGENT_MODEL`, `TRADER_AGENT_MODEL`).
- b) Все на `anthropic/claude-sonnet-4.6` — лучше качество, в ~5 раз дороже.
- c) Микс: PRICE/NEWS — Haiku, TRADER — Sonnet (на финальном решении).
- d) Своё: Выстави в `.env.example` default deepseek (для каждой модели своё поле), я затем сменю на необходимые

Ответ: d

### 5.2. Эмбеддинги новостей — какая модель и размерность?

- a) **[рекомендую]** `openai/text-embedding-3-small` через OpenRouter — 1536 dim, дёшево, хорошее качество для семантического поиска. pgvector колонка `vector(1536)`.
- b) `openai/text-embedding-3-large` — 3072 dim, в 6 раз дороже, выигрыш на коротких новостях минимален.
- c) Локальная модель (BAAI/bge-small) через `sentence-transformers` — без внешнего API, нужен GPU/RAM.
- d) Своё —

Ответ: a

### 5.3. Поведение при ошибке LLM-вызова (timeout/429/5xx)?

- a) **[рекомендую]** Ретрай 4 раза с экспоненциальной паузой (1s/3s/9s/27s). Если всё ещё ошибка — запись в БД с `ERROR`, шаг pipeline для этой монеты пропускается, в телеграм идёт уведомление.
- b) Один ретрай, на втором падении весь pipeline-тик прерывается.
- c) Бесконечный ретрай до успеха.
- d) Своё —

Ответ: a

### 5.4. Промпты агентов — где хранить?

- a) **[рекомендую]** В отдельной папке `app/services/agents/prompts/*.md`, версионируются вместе с кодом. Промпты на русском, ответы агентов — на русском.
- b) Прямо в `config.py` как строки.
- c) В БД (можно менять на лету) — оверкилл для MVP.
- d) Своё —

Ответ: a

## 7. Расписание и режимы запуска

- b) Системный cron в docker-контейнере.
- c) Celery beat — оверкилл.
- d) Своё —

Ответ: a

### 7.2. Переключение режимов через `.env`?

- a) **[рекомендую]** `SCHEDULER_MODE=cron` (4 раза в сутки 00/06/12/18 UTC) или `SCHEDULER_MODE=interval` + `SCHEDULER_INTERVAL_MINUTES=N`. В `interval` первый запуск сразу при старте сервиса.
- b) Только cron, без режима interval.
- c) Только interval.
- d) Своё —

Ответ: a

### 7.3. Обработка криптовалют в pipeline — параллельно или последовательно?

- a) **[рекомендую]** Последовательно (BTC → ETH → TON). Проще логировать, меньше шанс «гонок» по балансу. Все агенты внутри одной монеты — где можно — параллельно (PRICE и NEWS ветки идут одновременно).
- b) Параллельно все 3 монеты через `asyncio.gather` — быстрее, но баланс делить сложнее.
- c) Своё —
